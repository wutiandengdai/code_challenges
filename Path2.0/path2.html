Path2.0 Coding Challenge

You are given an R by C matrix consisting of booleans that represents a board. Each True boolean represents a wall. Each False boolean represents a tile you can walk on. Given this matrix with an unknown start coordinate and an unkown end coordinate return the number of steps required to reach the end coordinate from the start. The start square is located in row 0 and the end coordiante is located on the last row. If there is no possible path, then return null. You can move up, left, down, and right. You cannot move through walls. You cannot wrap around the edges of the board.

For example, given the following board:
[[t, t, t, f, t],
[t, t, t, f, f],
[f, f, f, t, f],
[f, t, f, f, f],
[f, f, t, t, t],
[t, f, t, t, t]]

Find the path that will allow you to exit on the other side. Start = (0, 1)(bottom row second column) and end = (5, 3)(top row 4th column), the number of steps required to reach the end is 13, since we would need to go through (1, 1) because there is a wall everywhere else on the second row. (Note* the displayed grid is inverted for display purposes so that row 0 is on the bottom)
